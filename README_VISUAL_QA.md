# Визуальный QA — Travel Territory

Сайт содержит анимации и WebGL, поэтому для проверки и скриншотов
предусмотрен режим заморозки.

## Запуск

```bash
npm run dev            # http://localhost:3000
npm run dev -- -p 3001 # если порт 3000 занят → http://localhost:3001
```

## Режимы (query-параметры)

| Параметр | Что делает |
| --- | --- |
| `?qa=1` | Полный QA-режим: сцена замирает, Lenis выключен, анимации остановлены |
| `?scene=freeze` | Замораживает только 3D-сцену в хорошем кадре |
| `?motion=off` | Отключает анимации (как `prefers-reduced-motion`) |

Параметры комбинируются: `/?qa=1&motion=off&scene=freeze`.

## Что проверять

### 1. Главная и 3D-сцена
Откройте `/`. Глобальная 3D-сцена должна быть фоном; при скролле камера
проходит 8 стадий (герой → подбор → горящие туры → страны → почему мы →
как это работает → отзывы → контакты). HUD слева внизу показывает прогресс.

### 2. Замороженный режим
Откройте `/?qa=1&motion=off&scene=freeze`. Сцена и анимации останавливаются,
вёрстка остаётся идентичной — можно делать скриншоты любой секции.

### 3. Мобильная версия
Уменьшите окно до ширины ~390 px (DevTools → toggle device toolbar).
Проверьте: нет горизонтального скролла, меню-бургер работает, кнопки кликабельны.

### 4. Reduced-motion
Включите в системе «Уменьшить движение» **или** откройте `?motion=off`.
Анимации появления должны исчезнуть, контент — остаться видимым.

### 5. Скриншоты
Лучше всего в режиме `/?qa=1&motion=off&scene=freeze` — кадр статичен.
Для секций ниже первого экрана просто прокрутите к ним: камера сцены
встанет в кадр соответствующей стадии.

## Если 3D не рендерится

- 3D-сцена грузится отдельным чанком (`dynamic`, `ssr:false`) — при загрузке
  и при отсутствии WebGL показывается премиальная не-3D заглушка `SceneFallback`.
- Проверьте поддержку WebGL: `chrome://gpu`.
- Ошибки WebGL перехватываются `SceneErrorBoundary` → сайт остаётся рабочим.
- Консоль браузера — основные сообщения об ошибках сцены выводятся туда.
